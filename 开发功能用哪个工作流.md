### 1. 核心推荐：直接用于"开发功能"
- **`affaan-m/everything-claude-code` 
	- 最为灵活
- **`obra/superpowers` (用于复杂功能)**
    - **是什么**：这是一套**高级技能框架（Agentic Skills Framework）**。它不仅仅是写代码，更是教 AI 遵循资深工程师的开发流程（Methodology）。
    - **特点**：强制执行 **TDD（测试驱动开发）**、任务拆解（把大功能拆成 5 分钟的小任务）、计划先行、以及 Sub-agent（子智能体）协作。
    - **适用场景**：如果你希望 AI 写出的代码**质量高、有测试覆盖、逻辑严密**，并且不介意它花更多时间在思考和规划上，选这个。它能防止 AI "写出只有看起来能跑的垃圾代码"。
    - **如何用**：它通常作为一个插件安装到 Claude Code 中。
- **`anthropics/claude-code/.../feature-dev` (官方标准)**
    - **特点**：作为官方实现，它与 Claude Code 的原生体验结合最好。它定义了标准的 探索 (Explore) -> 设计 (Design) -> 实施 (Implementation) -> 评审 (Review/Verify) 流程。
    - **适用场景**：如果你想要**最标准、最原生**的体验，或者刚开始上手 Claude Code，这是一个最稳妥的起点。
    - “评审/验证”： 主要是**“同行评审”（Peer Review）。
        - 启动一个专门的 Reviewer Agent，这个 AI 助手会阅读代码，从静态逻辑、代码风格、安全性角度给你提建议。 
        - 问题所在：“纸上谈兵”。虽然它也会尝试运行一些命令，但它的核心指标是 “AI 觉得没问题”。如果代码运行报错了，它可能也只是建议你去修。
### 2. 辅助工具：用于"增强体验"或"管理流程"

另外两个项目更多是作为**资源库**或**工作流工具**，而不是直接的"功能开发代理"。
- **是什么**：这是一个**配置和资源的合集**。作者（黑客松赢家）把自己用了 10 个月的最佳配置、Skills、MCP 服务、Prompts 等都打包在了一起。
- **作用**：它相当于一个"增强包"。你可以从中挑选好用的 Skill（比如专门写 Django 的 Skill，或者专门做代码审查的 Skill）来辅助你的开发。
- **建议**：**不需要二选一**。你可以配合 Superpowers 或官方插件主要使用，但从这里面“淘”一些好用的具体技能（Skill）来增强你的 AI。

    - **适用场景**：如果你觉得频繁的 git 操作很繁琐，或者想要通过 AI 自动化管理 PR 和 Commit，这个工具会很有用。

### 总结建议

1. **首选方案 (如果你追求高质量)**： 使用 **`obra/superpowers`**。它提供的 TDD 和结构化工作流是目前 AI 辅助编程中非常先进的理念，能帮你写出更健壮的代码。
2. **备选方案 (如果你追求简单/原生)**： 使用 **`feature-dev` (官方插件)**。这是最保险的选择。
3. **搭配使用**： 无论你选上面哪个，都可以去 **`everything-claude-code`** 里看看有没有适合你项目技术栈（比如 React, Python 等）的特定 **Skill**，把它加到你的配置里。---
